### 1.학습 후기
RESTful API에 대해 아예 모르고 그냥 할 때에는 그냥 대충 느낌으로 이떈 이런 거구나 하고 개념은 몰랐는데 개념을 공부하면서 더 이해가 잘 되게 되었습니다.
### 2. 핵심 키워드 정리

#### 1) RESTful api란?
    RESTful API란 REST: Representational State Transfer의 약자로 웹 기반 통신에서 자원에 대한 주소를 사용해 정보를 주고받는 방법입니다.
    주로 HTTP프로토콜을 이용하여 데이터를 송수신하며 자원의 상태를 주고받는 모든 것을 의미하고 자원의 상태를 HTTP Method를 통해 관리합니다.
    RESTful API란 REST의 원리를 따르는 시스템을 의미하지만 REST를 사용한다고 모두 RESTful 하진 않고 REST API의 설계 규칙을 올바르게 지킨 시스템을 RESTful하다고 할 수 있습니다.
    REST의 특징으로는 다음이 있습니다.
        1. 서버-클라이언트 구조
        2. 무상태
        3. 캐시 처리 가능
        4. 계층화
        5. 인터페이스 일관성

#### 2) 멱등성이란?
    동일한 연산을 수행해도 결과가 달라지지 않는 성질입니다.
    멱등성의 실제 적용 사례로는 은행 거래 시스템이 있습니다. 예를 들어 사용자가 계좌 이체를 요청 할 때 동일한 트랜잭션 아이디로 여러 번 요청이 들어와도 시스템은 단 한 번만 요청을 처리해야 합니다.
    멱등성을 확보하기 위해서 가장 일반적인 방법으로는 트랜잭션 아이디나 타임스탬프 등의 유니크한 식별자를 사용하는 방법이 있고 데이터베이스의 고유성 제약 조건, 잠금 메커니즘, 타임아웃 등 다양한 기술적 방법을 통해서도 확보 할 수 있습니다.

#### 3) HTTP 메서드 종류
    GET : 리소스 요청
    POST : 내용 전송 (HTTP 바디를 통해 전송)
    HEAD : 메세지 헤더 취득 (GET과 비슷하나 응답 메세지에 body가 없이 HTTP 헤더만 보냄)
    PUT : 내용 갱신/파일 전송, 기존의 데이터를 전부 교체
    DELETE : 파일 삭제
    OPTIONS : 가능한 메소드 옵션에 대한 질의
    TRACE : 요청 리소스가 수신되는 경로를 보여줌
    CONNECT : 프록시 서버를 통해 서버까지 직접 연결 통로를 생성하는 요청
    PATCH : 내용 갱신, 기존의 데이터는 유지하고 부분 교체
