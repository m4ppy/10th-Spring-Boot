# Chapter02_API 설계하기 - API 명세서, RESTful API

## 1.학습후기
 - 혼자 스프링부트를 공부할 때는 그저 대충 EndPoint를 설정하고 오고가는 내용은 혼자 알고 있으면 되는정도로 생각했었는데
협업에 관점에서 보면 Request Body, Request Header, 등과 같은 세세한 항목들을 문서화를 해놓아야 다른 사람들이 이해하기 편하고 소통하는데에도 많은 도움이 될 것 같다는 생각을 했고 이러한 학습은 향후에 협업을 하는 과정에서 많은 도움이 될 것 이라고 생각을 한다. 
## 2.핵심 키워드 정리

### RESTful API란?

- HTTP를 기반으로 하는 웹 서비스 아키텍처이며
HTTP 메서드(CRUD)와 자원을 이용해 서로 간의 통신을 주고받는 REST의 설계 규칙을 잘 지켜서 설계된 API를 의미합니다.

### 멱등성이란?

- 멱등성이란 동일한 요청을 한번 또는 수십 번 시도해도 같은 효과와 서버의 상태가 동일 하게 남는 특성을 의미하며 CRUD에서 멱등성을 가지는 메서드 GET, PUT, DELETE이며 PATCH, POST는 멱등성을 지니지 않습니다.

### HTTP메서드 종류

- HTTP메서드는 CRUD(생서,조회,갱신,삭제)로 구분됩니다.
- GET: get 요청에서 서버에 데이터를 전달하는 경우, 쿼리 파라미터를 통해 전달합니다. 이때 쿼리 파라미터는 무방비하게 노출됩니다.
- POST: post 요청은 데이터를 처리하고, 메세지 바디를 통해서 데이터를 전달합니다.
  주로 신규 리소스를 생성, 등록, 업데이트 하는데 사용되며 쿼리 파라미터로는 외부로 노출되도 상관없는 값을 전송하지만 리퀘스트 바디(응답내용)로는 민감한 정보를 담아 전송합니다.
- PUT: put 요청은 리소스가 있으면 대체하고 없으면 생성합니다. 만약 대체 시 전체 내용을 갱신합니다.
- PATCH: patch는 put과 같이 리소스를 수정하는 역할을 하지만, 리소스를 부분 변경한다는 점에서 차이가 있습니다.
- DELETE: 리소스를 제거할 때 사용되며 Hard Delete와 Soft Delete가 있습니다.
    - Hard Delete는 데이터를 바로 삭제하는 방식으로 복구가 힘드나 용량을 절약할 수 있습니다.
    - Soft Delete방식은 데이터를 삭제했다 라는 표시만을 남깁니다.

## 3.미션
![chapter2명세서.png](images/chapter2%EB%AA%85%EC%84%B8%EC%84%9C.png)
### 미션목록조회

- 특정 사용자의 미션 목록 데이터를 서버에서 불러오는 기능입니다. URL 끝에 쿼리 파라미터(`status=CHALLENGING` 또는 `COMPLETE`)를 덧붙여 미션의 진행 상태에 따라 원하는 목록만 필터링하여 조회할 수 있습니다. 헤더에 포함된 인증 토큰을 통해 정상적인 사용자 요청인지 확인합니다.

### 홈 화면

- 로그인한 사용자가 서비스의 메인 홈 화면에 진입했을 때 보여줄 초기 맞춤형 데이터를 서버에 요청하는 기능입니다.

### 미션성공누르기

- 사용자가 수행 중인 미션을 완료 상태로 변경할 때 사용하는 기능입니다. 데이터의 전체가 아닌 '상태(status)'라는 특정 항목만 수정하기 때문에 `PATCH` 메서드를 사용합니다. 본문(Body)에 변경할 상태 값을 담아 보내며, 서버가 이 데이터의 형식을 정확히 해석할 수 있도록 `Content-Type: application/json` 헤더를 함께 전송합니다.

### 리뷰작성

- 미션을 완료한 후 사용자가 해당 미션에 대한 새로운 리뷰 데이터를 서버에 생성하는 기능입니다. 주소의 경로 변수(Path Variable)를 통해 어떤 미션에 대한 리뷰인지 식별하며, 본문에 사용자가 입력한 별점(`score`)과 내용(`content`)을 JSON 형태로 담아 전송합니다.

### 마이페이지

- 특정 사용자의 개인 정보, 포인트, 활동 내역 등 마이페이지 화면을 구성하는 데 필요한 데이터를 서버에 요청하여 가져오는 기능입니다.

### 회원 가입 하기

- 서비스에 새로운 사용자를 등록(생성)하는 기능입니다. 아직 로그인을 하기 전 단계이므로 다른 API들과 달리 인증 토큰 헤더가 필요하지 않습니다. 가입에 필요한 이메일, 비밀번호, 이름, 전화번호 등의 필수 정보를 본문에 담아 서버에 전송합니다.
